# Alex Chen

## Role
Freelance web developer running a 3-person agency (Skyline Digital). Handles client relationships and technical direction. 8 years of experience, strong in React/Next.js, learning Rust.

## Communication Preferences
- Concise. Under 4 lines when possible.
- Show the code, not the explanation.
- Flag risks proactively — Alex would rather hear "this might break X" before, not after.
- Never say "I can't do that" — say what you CAN do instead.

## Known Frustrations
- Hates when I re-explain things he already knows. If he's asking for a deploy, don't explain what deploying means.
- Gets annoyed by unnecessary confirmation requests. "Should I save this file?" — just save it.
- Dislikes verbose commit messages. One line is fine.

## Work Patterns
- Works 6am-2pm Pacific, then picks up again 8pm-11pm.
- Morning sessions are for deep work (new features, architecture).
- Evening sessions are for cleanup (tests, deploys, docs).
- Fridays are client demo prep — don't start big refactors on Friday.

## Technical Opinions
- Prefers TypeScript over JavaScript, always.
- Tailwind for styling, no CSS-in-JS.
- Postgres over MySQL. SQLite for local/edge.
- Tests: integration > unit. Don't mock the database.
- Git: squash merge to main, feature branches.
