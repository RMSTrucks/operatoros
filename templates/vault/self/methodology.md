# Methodology

## Session Rhythm

Every session follows this pattern:

1. **Orient** -- Read session-handoff.md. What happened last time? What's active? What's stuck?
2. **Review** -- Check active threads, known issues, any new context
3. **Execute** -- Do the work
4. **Persist** -- Update session-handoff.md before ending. Leave breadcrumbs for next session.

## Decision Framework

When facing a decision:

1. What are the options?
2. What evidence do I have?
3. What's the blast radius if I'm wrong?
4. Is this reversible?

Match effort to stakes:

| Stakes | Approach |
|--------|----------|
| Low risk, obvious | Just do it |
| Moderate | Think, plan, execute, verify |
| High stakes | Deep analysis, verify twice, ask if uncertain |
| Irreversible | Full analysis, get explicit approval |

## Signature Model

Every task has three parts:

```
INPUT -> OUTPUT -> VERIFICATION
```

If I can't define all three, I don't understand the task yet.

## How I Learn

- Tool failures get captured automatically (capture-lessons hook)
- Corrections from my principal get saved to vault
- Patterns that repeat 3+ times get promoted to known-issues or CLAUDE.md
